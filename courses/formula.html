Formula

You should no longer be a rookie and may need to use Formula to help you achieve more advanced requirements in Notion.

  • Author
  • STAR Chang

To make better use of Databases, it's essential to learn Formulas. Formulas are not only used for calculating numbers but also for determining rules, displaying statuses, showing dates and times, and much more. Mastering Formulas will open up a whole new universe of possibilities.

Let's start with a simple appetizer. When creating a Formula field in a Database, opening the field will reveal a PopUp with options like Formula Input, Properties, and Constants. The naming convention for Properties will be something like prop("example").

Looking at the example images, the "Remain" field represents the desired control over the current inventory of goods. To achieve this, we can use the formula prop("Amount") - prop("Sold") in the "Remain" field. This is a basic application of a Formula, and that's how Formulas generally work.

Formula Examples

Now that we have covered the basic functionality, let's dive into some examples to give you a clearer understanding of the various applications of Formulas. Although it may not be easy to grasp the examples right away, they are meant to pique your curiosity and motivate you to learn more about Constants.

To explore the available Constants and their descriptions, please scroll down to the section on Constants.

Introduction to the "if" structure

Allow me to introduce the structure of the "if" statement first. This will help you better understand the examples mentioned earlier.

if( Conditions , Yes , No )

A simple conditional expression structure is divided into three parts within parentheses, as follows:

  • Conditions: The conditions to be evaluated.
  • Yes: The result to be displayed if the conditions are met.
  • No: The result to be displayed if the conditions are not met.

Let's take a simple scenario to reinforce your understanding of the "if" statement.

If the restaurant is full, I will take the food to go; otherwise, if it is not full, I will dine in.

The actual formula to incorporate into the database would be written as follows, using the "Take out / Eat in" field.

 
if(prop("People")==prop("Seats"),"Take out","Eat in")
  • Conditions â†’ prop("People")==prop("Seats")
  • Yes â†’ "Take out"
  • No â†’ "Eat in"

As you progress, you will encounter more complex formulas that are mostly extensions of this basic structure.

Introduction to Constants

I usually write the following formulas in a more conventional way, avoiding the use of mathematical symbols like +, -, *, and /. This makes it easier for you to understand the basic logic.

PS. You may often see values in some formulas enclosed in double quotes, while others are not, such as "3" and 3. For the system, these are two completely different values. The ones enclosed in double quotes represent "strings," while those without double quotes represent "numeric values." This distinction is important to keep in mind.

Mathematical operations

  • add
    Addition or concatenation of strings
    3 + 4 = 7
    add(1, 3) == 4
    "add" + "text" == "addtext"
  • subtract
    Subtraction or string subtraction
    3 - 1 == 2
    subtract(4, 5) == -1
  • multiply
    Multiplication
    6 * 9 == 54
    multiply(2, 10) == 20
  • divide
    Division
    12 / 4 == 3
    divide(12, 3) == 4
  • pow
    Exponentiation
    5 ^ 3 == 125
    pow(2, 6) == 64
  • mod
    Remainder
    7 % 5 == 2
    mod(3, 3) == 0
  • unaryMinus
    Negative number
    -1 + 2 == 1
    unaryMinus(42) == -42

Condition operations

  • and
    Both are True, then it is True.
    true and false == false
    and(true, true) == true
  • or
    Either one of them is True, then it is True.
    false or true == true
    or(false, false) == false
  • equal
    The first one is equal to the second one, then it is True.
    (3 * 5 == 15) == true
    equal(false, not true) == true
  • unequal
    The first one is not equal to the second one, then it is True.
    (6 * 9 != 42) == false
    (true != not false) == false
  • larger
    The first one is greater than the second one, then it is True.
    5 > 3 == true
    larger(5, 3) == true
  • largerEq
    The first one is greater than or equal to the second one, then it is True.
    5 >= 3 == true
    4 >= 4 == true
    largerEq(5, 3) == true
    largerEq(4, 4) == true
  • smaller
    The first one is less than the second one, then it is True.
    10 < 8 == false
    smaller(10, 8) == false
  • smallerEq
    The first one is less than or equal to the second one, then it is True.
    10 <= 8 == false
    8 <= 8 == true
    smallerEq(10, 8) == false
    smallerEq(8, 8) == false
  • not
    The opposite of True and False.
    not(false) == true

String functions

  • concat
    Concatenate strings
    "dog" +"go" == "doggo"
    concat("dog", "go") == "doggo"
  • replace
    Replace the first occurrence of a substring
    replace("1-2-3", "-", "!") == "1!2-3"
    replace("abcc abcccc", "abc{2,}", "!") == "! abcccc"
    replace("123333 123333", "123", "!") == "!333 123333"
    If there are duspanlicate strings, the one appearing earlier takes precedence.

  • replaceAll
    Replace all occurrences of a substring
    replaceAll("1-2-3", "-", "!") == "1!2!3"
    replaceAll("abcc abcccc", "abc{2,}", "!") == "! !"
  • length
    Calculate the length of a string
    length("Hello world") == 11
  • contains
    Returns true if a string or value is present
    contains("notion", "ion") == true
  • test
    Returns true if a string is present
    test("1-2-3", "-") == true
  • format
    Extract values from a string
    format(42) == "42"
    format(true) == "true"
    format(now()) == "Wed Dec 31 1969
  • join
    Concatenate a character at the beginning of subsequent characters
    join("-", "a", "b", "c") == "a-b-c"
  • slice
    Remove characters
    slice("Hello world", 1, 5) == "ello"
    slice("notion", 3) == "ion"

Date functions

  • hour
    Extract hour
    hour(now()) == 17
  • minute
    Extract minute
    minute(now()) == 45
  • month
    Extract month
    month(now()) == 11
  • year
    Extract year
    year(now()) == 1984
  • date
    Extract day of the date
    date(now()) == 13
  • day
    Extract day of the week
    day(now()) == 3
  • now
    Current date and time
    now() == Feb 2, 1996 6:30 PM
  • dateAdd
    Add a specified duration to a date and time
    dateAdd(date, amount, "years")
    dateAdd(date, amount, "quarters")
    dateAdd(date, amount, "months")
    dateAdd(date, amount, "weeks")
    dateAdd(date, amount, "days")
    dateAdd(date, amount, "hours")
    dateAdd(date, amount, "minutes")
    dateAdd(date, amount, "seconds")
    dateAdd(date, amount, "milliseconds")
    date â‡¢ Specifies a date/calendar field.

    amount â‡¢ Fill in a number to indicate how much to add in the future, for example, adding 2 years, adding 2 quarters, adding 2 minutes.

  • dateSubtract
    Subtract a specified duration from a date and time
    dateSubtract(date, amount, "years")
    dateSubtract(date, amount, "quarters")
    dateSubtract(date, amount, "months")
    dateSubtract(date, amount, "weeks")
    dateSubtract(date, amount, "days")
    dateSubtract(date, amount, "hours")
    dateSubtract(date, amount, "minutes")
    dateSubtract(date, amount, "seconds")
    dateSubtract(date, amount, "milliseconds")
    date â‡¢ Specifies a date/calendar field.

    amount â‡¢ Fill in a number to indicate how much to subtract in the past, for example, subtracting 2 years, subtracting 2 quarters, subtracting 2 minutes.

  • dateBetween
    Calculate the difference between two dates
    dateBetween(date, date2, "years")
    dateBetween(date, date2, "quarters")
    dateBetween(date, date2, "months")
    dateBetween(date, date2, "weeks")
    dateBetween(date, date2, "days")
    dateBetween(date, date2, "hours")
    dateBetween(date, date2, "minutes")
    dateBetween(date, date2, "seconds")
    dateBetween(date, date2, "milliseconds")
    date â‡¢ Specifies a date/calendar field.

    date2 â‡¢ Specifies a date/calendar field.

  • start
    Start of a date and time range
    start(prop("Date")) == Feb 2, 1996
  • end
    End of a date and time range
    end(prop("Date")) == Feb 2, 1996
    For example, the end of "2022/12/01 â†’ 2022/12/20" is 2022/12/20.

  • formatDate
    Display date format
    formatDate(now(), "MMMM D YYYY, HH:mm") == March 30 2010, 12:00
    formatDate(now(), "YYYY/MM/DD, HH:mm") == 2010/03/30, 12:00
    formatDate(now(), "MM/DD/YYYY, HH:mm") == 03/30/2010, 12:00
    formatDate(now(), "HH:mm A") == 12:00 PM
    formatDate(now(), "M/D/YY") == 3/30/10
    formatDate(now(), "ww") == 50
    The "now()" can also be used to specify a date field.

  • fromTimestamp
    Add milliseconds starting from the system start date
    fromTimestamp(2000000000000) == Tue May 17 2033
    The starting date and time is January 1, 1970, at 08:00.

    The unit of measurement for numbers is milliseconds.

  • timestamp
    Calculate the number of milliseconds added from the system start date
    timestamp(now()) == 1512593154718
    The starting date and time is January 1, 1970, at 08:00.

    The unit of measurement for numbers is milliseconds.
